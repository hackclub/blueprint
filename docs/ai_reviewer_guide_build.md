# Build Review Checklist

This is the post-build check — verifying the project was actually built and works.

## Physical build verification

- [ ] Photos of real, physically built hardware exist (not just renders or 3D models)
- [ ] The project functions as described
- [ ] Final photos show the completed project from multiple angles
- [ ] The build matches the design — components, wiring, and enclosure match what was planned

## Design review follow-up

- [ ] Issues flagged in the design review have been addressed
- [ ] Any changes from the original design are documented

## JSON check names for this phase

Use these keys in the `checks` object of your JSON output:

`build_photos_exist`, `project_functions`, `final_photos_complete`, `build_matches_design`, `design_issues_addressed`, `changes_documented`, `no_ai_content`, `original_design`
